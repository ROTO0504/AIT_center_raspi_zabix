# テストコマンド集

`ver2/test_mail_scenarios.py` を使うと、実メールや Microsoft Graph に接続せずに、判定ロジックと GPIO 動作をシナリオ別に確認できます。

`--list` は依存未導入でも実行でき、各シナリオ実行時も `msal` は不要です。

## 事前準備

リポジトリ直下で実行します。

```bash
cd /Desktop/raspi
```

必要なら依存を入れます。

```bash
python3 -m pip install -r ver2/requirements.txt
```

Raspberry Pi 実機で LED / ブザーを確認する場合は、本番サービスと GPIO 競合しないように先に止めてください。

```bash
sudo systemctl stop mail2alertlight
```

テスト後に戻すコマンド:

```bash
sudo systemctl restart mail2alertlight
```

## 使い方

### シナリオ一覧

```bash
python3 test_mail_scenarios.py --list
```

### JSON で見たい場合

```bash
python3 test_mail_scenarios.py high_problem --json
```

## シナリオ別コマンド

### 1. メール 0 件で初回判定

```bash
python3 test_mail_scenarios.py startup_empty
```

期待値:

- `overall=normal`
- 赤 LED OFF
- ブザーなし

### 2. high 障害メール受信

```bash
python3 test_mail_scenarios.py high_problem
```

期待値:

- `overall=high`
- 赤 LED ON
- ブザー 2 回
- 理由に更新内容が出る

### 3. disaster 障害メール受信

```bash
python3 test_mail_scenarios.py disaster_problem
```

期待値:

- `overall=high`
- 赤 LED ON
- ブザー 2 回

### 4. warning メールのみ受信

```bash
python3  test_mail_scenarios.py warning_only
```

期待値:

- `overall=normal`
- 赤 LED OFF
- ブザーなし

### 5. high 障害のあと復旧メール受信

```bash
python3 test_mail_scenarios.py resolved_after_problem
```

期待値:

- 最終的に `overall=normal`
- 赤 LED OFF
- `host_metrics` に復旧時間が入る

### 6. warning と high が混在

```bash
python3 test_mail_scenarios.py mixed_hosts
```

期待値:

- high が 1 台でもあれば `overall=high`
- 赤 LED ON

### 7. 判定対象外メール

```bash
python3  test_mail_scenarios.py ignored_mail
```

期待値:

- 状態変化なし
- `overall=normal`
- 赤 LED OFF

### 8. high 障害中にボタンを押して消灯

```bash
python3 test_mail_scenarios.py high_then_button
```

期待値:

- `overall=high` のまま
- ボタン押下直後に赤 LED OFF
- `silenced=true`
- ブザー履歴は「high受信の2回 + ボタン押下の1回」
- 次の判定では理由に `（ボタンで消灯中）` が付く

## よく使う確認コマンド

### 1 本ずつ順番に試す

```bash
python3 test_mail_scenarios.py startup_empty
python3 test_mail_scenarios.py warning_only
python3 test_mail_scenarios.py high_problem
python3 test_mail_scenarios.py high_then_button
python3 test_mail_scenarios.py resolved_after_problem
python3 test_mail_scenarios.py ignored_mail
```

### 実行結果を保存

```bash
python3 test_mail_scenarios.py high_problem --json | tee high_problem_result.json
```

### サービス停止から high テスト、サービス復帰までまとめて実行

```bash
sudo systemctl stop mail2alertlight && \
python3 test_mail_scenarios.py high_problem && \
sudo systemctl restart mail2alertlight
```

## 補足

- このテストは **ダミーメール** を使うため、ネットワーク未接続でも実行できます。
- Raspberry Pi 実機では、GPIO が有効なら実際に赤 LED / ブザーが動きます。
- Mac など GPIO がない環境では、出力結果の `gpio_available: no` で確認できます。
- 現在のロジックでは、赤 LED が点灯するのは `Severity: high` または `Severity: disaster` のときです。
